# POWRAY-UNI Uploader V1.0 사용 설명서


## 개요

`POWRAY-UNI_Uploader_V1.0.exe`는 POWRAY-UNI 장비의 APP 펌웨어를 RS485 또는 W-DMX로 업로드하기 위한 Windows 프로그램이다.

RS485는 장비 응답을 확인하는 1:1 또는 순차 다중 업로드 방식이다. W-DMX는 장비 TX 응답이 없는 RX-only 환경이므로 DMX512 프레임을 반복 송신하는 1:N 방식으로 동작한다.

## 실행 파일 생성

빌드 배치파일:

```text
Bootloader\Tools\build_rs485_uploader.bat
```

배치파일을 실행하면 실행한 경로에 아래 파일이 생성된다.

```text
POWRAY-UNI_Uploader_V1.0.exe
```

빌드 완료 후 `build`, `dist`, `.spec`, `__pycache__`는 배치파일에서 삭제한다.

## 통신 속도

| 동작 | 속도 |
| --- | ---: |
| APP 동작 중 RS485 업로드 모드 요청 | 9600 bps, 8N1 |
| RS485 부트로더 업로드 | 921600 bps, 8N1 |
| W-DMX 업로드 모드 요청/업로드/종료 | 250000 bps, 8N2 |

## 버튼 기능

### RS485

| 버튼 | 기능 |
| --- | --- |
| RS485 업로드 모드 요청 | APP 동작 중 9600 bps로 32바이트 업로드 요청 패턴을 전송한다. |
| RS485 업로드 모드 종료 | 체크된 ID 장비에 RUN 명령을 보내 APP 복귀를 요청한다. 실패 시 ID별 1회 재시도한다. |
| RS485 업로드 | 체크된 ID 장비를 오름차순으로 1대씩 순차 업로드한다. 실패 시 ID별 1회 재시도 후 다음 ID로 진행한다. |

### W-DMX

| 버튼 | 기능 |
| --- | --- |
| W-DMX 업로드 모드 요청 | DMX512 250000 bps 8N2 프레임으로 업로드 요청 패턴을 반복 송신한다. |
| W-DMX 업로드 모드 종료 | DMX512 RUN 프레임을 반복 송신해 업로드 모드 종료를 요청한다. |
| W-DMX 업로드 | TX 응답 없이 DMX512 START/WRITE/COMPLETE 프레임을 반복 송신한다. |

## RS485 업로드 절차

1. PC와 장비를 RS485 A/B 라인으로 연결한다.
2. COM 포트를 선택한다.
3. 장비가 APP으로 동작 중이면 `RS485 업로드 모드 요청`을 누른다.
4. 장비가 리셋되고 LCD에 `WAITING UPLOAD`가 표시되는지 확인한다.
5. 검색 시작 ID와 검색 종료 ID를 지정한다.
6. `장비 검색`을 눌러 부트로더 모드 장비를 검색한다.
7. 업로드할 ID만 체크한다.
8. HEX 파일을 선택하거나 드래그앤드롭한다.
9. `RS485 업로드`를 누른다.
10. 업로더는 체크된 ID를 오름차순으로 순차 업로드한다.
11. 특정 ID에서 통신 오류가 발생하면 1회 재시도한다.
12. 재시도 후에도 실패한 ID는 로그에 기록하고 다음 ID로 진행한다.
13. 로그에 업로드 완료 또는 성공/실패 요약이 표시되면 결과를 확인한다.

## RS485 업로드 모드 종료

업로드 요청 후 실제 업로드하지 않고 APP으로 복귀하려면 다음 순서로 진행한다.

1. 장비 LCD가 `WAITING UPLOAD` 상태인지 확인한다.
2. `장비 검색`을 하거나 수동 ID 추가로 종료할 ID를 목록에 추가한다.
3. 종료할 ID를 체크한다.
4. `RS485 업로드 모드 종료`를 누른다.
5. 업로더가 체크된 ID에 RUN 명령을 전송한다.
6. 실패한 ID는 1회 재시도하고, 다시 실패하면 다음 ID로 진행한다.

## W-DMX 업로드 절차

1. PC 송신 장치와 W-DMX 경로를 연결한다.
2. COM 포트를 선택한다.
3. 장비가 APP으로 동작 중이면 `W-DMX 업로드 모드 요청`을 누른다.
4. 장비가 부트로더 업로드 모드로 진입했는지 확인한다.
5. HEX 파일을 선택하거나 드래그앤드롭한다.
6. `W-DMX 업로드`를 누른다.
7. 업로더는 START, WRITE, COMPLETE 프레임을 반복 송신한다.
8. W-DMX는 장비 응답이 없으므로 PC에서는 송신 완료 여부만 확인할 수 있다.
9. 장비는 수신 완료 후 CRC/magic 검증에 성공하면 백업 저장 후 리부트한다.
10. 실패하면 유효한 백업 펌웨어를 APP 영역으로 복구한 뒤 리부트한다.

## HEX 파일 조건

- Intel HEX 형식이어야 한다.
- APP 시작 주소는 `0x08010000`이어야 한다.
- APP 영역은 `0x08010000~0x0807FFFF` 범위 안에 있어야 한다.
- `0x08000000` 기준 HEX는 부트로더 영역과 충돌하므로 사용하면 안 된다.
- `0x08010200~0x08010213` 영역은 부트로더 검증용 magic/size/crc/version 메타 영역으로 예약되어야 한다.
- 업로더는 송신 직전에 메타 영역과 이미지 마지막 8바이트 magic, APP CRC, 펌웨어 버전을 자동 삽입한다.
- 파일명에 `V1_4_2` 또는 `V1.4.2` 형식이 있으면 업로더가 버전을 자동 파싱한다.

## 오류 대처

| 증상 | 확인 사항 |
| --- | --- |
| COM 포트가 보이지 않음 | RS485 변환기 연결 후 `포트 새로고침`을 누른다. |
| RS485 업로드 모드 요청 후 반응 없음 | APP RS485 기본 속도가 9600 bps인지 확인한다. |
| 장비 검색 결과 없음 | 장비가 `WAITING UPLOAD` 상태인지, ID 범위가 맞는지 확인한다. |
| ID가 `[경고]`로 표시됨 | 동일 ID 장비가 중복 연결되었거나 응답이 중복 수신된 상태다. |
| `serial receive timeout` | 통신선, GND, RS485 A/B 극성, 장비 전원, 선택 ID를 확인한다. 업로더는 해당 ID를 재시도 후 다음 ID로 진행한다. |
| `APP linker must reserve boot metadata area` | APP을 `STM32F407VGTX_FLASH.ld` 기준으로 클린 빌드해 메타 영역을 예약해야 한다. |
| W-DMX 업로드 후 FAIL | W-DMX 수신 품질, START Code, DMX512 250000 bps 8N2 설정을 확인한다. |

## 주의 사항

- RS485 다중 업로드는 체크된 ID를 순차 처리한다.
- W-DMX는 RX-only 특성상 장비 검색과 ACK 확인이 불가능하다.
- W-DMX는 1:N 송신 구조이므로 같은 W-DMX 수신 범위의 장비가 모두 동일 펌웨어를 수신할 수 있다.
- 업로드 중 전원을 차단하지 않는다.
- 정상 검증된 APP만 백업 영역에 저장된다.
