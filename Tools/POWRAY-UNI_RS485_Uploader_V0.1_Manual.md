# POWRAY-UNI RS485 Uploader V0.1 사용 설명서

## 개요

`POWRAY-UNI_RS485_Uploader_V0.1.exe`는 POWRAY-UNI 장비의 APP 펌웨어를 RS485로 업로드하기 위한 Windows용 프로그램이다.

## 통신 속도

| 동작 | 속도 |
| --- | ---: |
| APP 동작 중 업로드 모드 요청 | 9600 bps |
| 부트로더 펌웨어 업로드 | 921600 bps |

APP이 9600 bps로 동작 중이므로 `업로드 모드 요청`만 9600 bps로 전송한다. 장비가 리셋되어 부트로더에 진입한 이후의 실제 업로드는 921600 bps로 진행한다.

## 실행 파일 생성

빌드 배치파일:

```text
Bootloader\Tools\build_rs485_uploader.bat
```

배치파일을 실행하면 실행한 경로에 아래 파일이 생성된다.

```text
POWRAY-UNI_RS485_Uploader_V0.1.exe
```

빌드 완료 후 `build`, `dist`, `.spec`, `__pycache__`는 자동 삭제된다.

## 업로드 전 준비

1. PC에 RS485 변환기를 연결한다.
2. 장비와 RS485 A/B 라인을 연결한다.
3. 업로드할 APP HEX 파일이 `0x08010000` 기준으로 빌드되었는지 확인한다.
4. `POWRAY-UNI_RS485_Uploader_V0.1.exe`를 실행한다.

## 기본 업로드 절차

1. COM 포트를 선택한다.
2. 장비가 APP으로 동작 중이면 `업로드 모드 요청`을 누른다.
3. 장비가 리셋되고 LCD에 `WAITING UPLOAD`가 표시되는지 확인한다.
4. 검색 시작 ID와 검색 종료 ID를 입력한다.
5. `장비 검색`을 눌러 지정한 범위의 부트로더 모드 장비 ID를 검색한다.
6. 검색된 ID는 오름차순으로 정렬되며 기본 체크 상태로 표시된다.
7. 같은 ID에서 2회 이상 응답이 수신되면 `[경고]`로 표시된다.
8. 업로드할 ID만 체크한다.
9. HEX 파일을 선택하거나 창에 드래그앤드롭한다.
10. `업로드`를 누른다.
11. 프로그램이 체크한 ID를 1대씩 순차 업로드한다.
12. 프로그래스바가 100%가 될 때까지 기다린다.
13. 로그창에 `업로드 완료`가 표시되면 완료이다.

## 업로드 요청 후 취소

업로드 모드 요청 후 펌웨어를 전송하지 않을 경우:

1. 장비 LCD에 `WAITING UPLOAD`가 표시된 상태인지 확인한다.
2. `장비 검색` 후 복귀시킬 ID만 체크한다.
3. `업로드 안하고 취소`를 누른다.
4. 프로그램이 921600 bps로 체크한 ID 장비에 `RUN` 명령을 보낸다.
5. 유효한 APP이 있으면 장비가 APP으로 복귀한다.

## HEX 파일 조건

- Intel HEX 형식이어야 한다.
- APP 시작 주소는 `0x08010000`이어야 한다.
- 주소 범위는 `0x08010000`부터 `0x080FFFFF` 안에 있어야 한다.
- `0x08000000` 기준 HEX는 사용하면 안 된다. 부트로더 영역과 충돌한다.
- `0x08010200`부터 16바이트는 부트로더 APP 유효성 검사용 magic/size/crc 메타 영역으로 예약된다.
- 업로더는 전송 직전에 메타 영역과 이미지 마지막 8바이트 magic, APP boot CRC를 자동 삽입한다.
- 메타 영역을 예약하지 않은 기존 링커 산출 HEX는 업로더가 주소 충돌 오류로 차단한다.

## 오류 대처

| 증상 | 확인 사항 |
| --- | --- |
| COM 포트가 보이지 않음 | RS485 변환기 연결 후 `포트 새로고침` 클릭 |
| 업로드 모드 요청 후 반응 없음 | APP RS485가 9600 bps 상태인지 확인 |
| `bootloader ready failed` | 장비가 `WAITING UPLOAD` 상태인지, COM 포트가 맞는지 확인 |
| 장비 검색 결과 없음 | 장비 ID가 0~255 범위인지, target ID 프로토콜 부트로더인지 확인 |
| ID가 `[경고]`로 표시됨 | 동일 ID 장비 중복 또는 응답 중복 가능성 확인 |
| 업로드 중 timeout | RS485 배선, 변환기 방향 제어, 통신 품질 확인 |
| HEX 주소 오류 | APP 링커 시작 주소가 `0x08010000`인지 확인 |
| 완료 후 APP 복귀 안 됨 | VERIFY 실패, APP vector, magic/size/crc 유효성 확인 |

## 주의 사항

- 업로드 중 장비 전원을 끄지 않는다.
- 업로드 중 RS485 케이블을 분리하지 않는다.
- 부트로더 진입 후에는 자동 timeout으로 APP에 복귀하지 않는다.
- 업로드 없이 복귀하려면 반드시 `업로드 안하고 취소`를 사용한다.
- 기존 APP이 손상된 경우 취소 명령으로 복귀할 수 없으며 정상 HEX 업로드가 필요하다.
- 동일 RS485 버스에 같은 ID 장비가 2대 이상 있으면 응답 충돌이 발생할 수 있다.
